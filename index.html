QUESTION 2
Describe the important built-in modules in Node.js with examples and explain the NPM Registry workflow.
üìù 16 Marks
Introduction: Node.js comes with built-in modules (no installation needed!) that provide essential functionalities like file operations, creating servers, working with paths, handling events, and getting system information.

1. Five Important Built-in Modules (10 marks)

1. fs Module (File System) - 2 marks

Purpose: Read, write, delete files and folders

const fs = require('fs');

// Write to a file
fs.writeFileSync("output.txt", "Hello World!");

// Read from a file
const data = fs.readFileSync("output.txt", "utf8");
console.log(data);  // Output: Hello World!

Use Case: Saving user data, reading configuration files, logging

2. http Module (Web Server) - 2 marks

Purpose: Create web servers to handle HTTP requests

const http = require('http');

// Create a server
const server = http.createServer((req, res) => {
    res.end("Hello from Node.js Server!");
});

server.listen(3000);
console.log("Server running on port 3000");

Use Case: Building REST APIs, web applications

3. os Module (Operating System) - 2 marks

Purpose: Get system information like memory, CPU, platform

const os = require('os');

console.log("Free Memory:", os.freemem());
console.log("Total Memory:", os.totalmem());
console.log("Platform:", os.platform());
console.log("CPU Info:", os.cpus());

Use Case: System monitoring, performance checking

4. path Module (File Paths) - 2 marks

Purpose: Work with file and directory paths

const path = require('path');

console.log(path.extname("file.txt"));      // Output: .txt
console.log(path.basename("C:/user/file.txt")); // Output: file.txt
console.log(path.dirname("C:/user/file.txt"));  // Output: C:/user
console.log(path.join("/user", "docs", "file.txt")); // C:/user/docs/file.txt

Use Case: Building file paths, getting file extensions

5. events Module (Event Handling) - 2 marks

Purpose: Create and manage custom events

const EventEmitter = require('events');
const emitter = new EventEmitter();

// Register event listener
emitter.on("start", () => {
    console.log("Application Started!");
});

// Trigger event
emitter.emit("start");  // Output: Application Started!

Use Case: Building event-driven applications, handling user actions

2. NPM Registry (3 marks)

What is NPM Registry? The NPM Registry is a central online database (npmjs.com) that stores all JavaScript packages. It's like a library where developers can publish their packages and others can download them.
Roles of NPM Registry:
  • Stores package metadata (name, version, author, description)
  • Distributes packages to developers worldwide
  • Tracks download statistics
  • Manages package versions (semantic versioning)
  • Ensures packages are available 24/7

Definition: A packaged module is a collection of code bundled together with its configuration (package.json) that can be reused in multiple projects or shared with other developers.
Benefits:

QUESTION 4
Demonstrate the use of setTimeout() function with a program that schedules multiple timeouts and explain its execution flow.
üìù 16 Marks
Introduction: setTimeout() is a built-in JavaScript function that executes code after a specified delay (in milliseconds). It's useful for scheduling tasks, creating delays, or simulating asynchronous behavior in applications.

1. What is setTimeout()? (2 marks)

Definition: setTimeout() is a function that schedules a callback function to run after a specified time delay.
Syntax: 
setTimeout(function, delayInMilliseconds);

// Example:
setTimeout(() => {
    console.log("Hello!");
}, 2000);  // Runs after 2 seconds

2. Program Objective (1 mark)

üéØ Goal: To create a program that schedules multiple timeouts at different intervals and displays messages showing how much time has elapsed since each was scheduled.

3. Complete Code Implementation (5 marks)

// Function to display timeout message
function simpleTimeout(ms) {
    console.log(`Timeout after ${ms} milliseconds.`);
}

// Start message
console.log("Scheduling timeouts...");

// Schedule multiple timeouts
setTimeout(() => simpleTimeout(1000), 1000);  // After 1 second
setTimeout(() => simpleTimeout(2000), 2000);  // After 2 seconds
setTimeout(() => simpleTimeout(3000), 3000);  // After 3 seconds
setTimeout(() => simpleTimeout(4000), 4000);  // After 4 seconds

console.log("All timeouts scheduled! Waiting for execution...");

6. How It Works - Detailed Explanation (2 marks)

Step 1 - Synchronous Execution: First, "Scheduling timeouts..." is printed immediately. Then all setTimeout() calls are registered (but not executed yet). Finally, "All timeouts scheduled!" is printed.
Step 2 - Asynchronous Execution: Node.js waits for the specified delays. After 1000ms, first timeout runs. After 2000ms, second runs, and so on. This is non-blocking - the program doesn't freeze while waiting.
Step 3 - Callback Execution: When each timeout's delay completes, its callback function (simpleTimeout) is executed with the appropriate millisecond value.
üí° Important Concepts:
  • Non-blocking: Code after setTimeout() runs immediately, doesn't wait
  • Asynchronous: Timeouts execute in the background
  • Callback: The function passed to setTimeout() is called later
  • 1000ms = 1 second: Delays are in milliseconds

üéØ Key Points to Remember:

  • setTimeout() schedules code to run after a delay
  • Delay is specified in milliseconds (1000ms = 1 second)
  • Code execution is non-blocking and asynchronous
  • Multiple setTimeout() can be scheduled simultaneously
  • Synchronous code runs first, then timeouts execute
QUESTION 5
Explain how to pass additional parameters to event listeners using anonymous functions with a complete working example.
üìù 16 Marks
Introduction: In Node.js, events normally accept limited parameters when emitted. However, we often need to pass extra information to event handlers. We can achieve this using anonymous wrapper functions (also called closures) that capture additional data.

1. Understanding the Problem (2 marks)

Basic Event Limitation: When you emit an event, you can only pass parameters that are specified in the emit() call. Sometimes you need additional context or data that isn't available at emit time.
Solution: Use an anonymous function (closure) to wrap your event handler. This function can access both the emitted parameters AND additional variables from its surrounding scope.

2. What is an Anonymous Function? (2 marks)

// Regular Named Function
function greet() {
    console.log("Hello");
}

// Anonymous Function (no name)
function() {
    console.log("Hello");
}

// Anonymous Function in Event Listener
emitter.on("event", function() {
    console.log("Event fired!");
});
üìå Definition: An anonymous function is a function without a name. It's often used as a callback or wrapper function. It can access variables from its outer scope (closure).

3. Complete Code Example (6 marks)

// Import EventEmitter class
const EventEmitter = require('events');

// Create an instance of EventEmitter
const emitter = new EventEmitter();

// Register event listener with anonymous wrapper function
emitter.on("login", (username) => {
    // Anonymous function with additional parameter
    (function(extraInfo){
        console.log("User:", username);
        console.log("Extra:", extraInfo);
    })("Logged in from Chrome Browser");
});

// Emit the event with username
emitter.emit("login", "Nazeeha");

4. Code Breakdown - Line by Line (4 marks)

Import EventEmitter 
const EventEmitter = require('events');

Imports the built-in events module which provides EventEmitter class for creating custom events.

Create Emitter Instance 
const emitter = new EventEmitter();

Creates a new event emitter object that can register listeners and emit events.

Register Event Listener 
emitter.on("login", (username) => {
    // This arrow function receives username from emit
});

Registers a listener for "login" event. The username parameter will be passed when event is emitted.

Anonymous Function with Extra Parameter 
(function(extraInfo){
    console.log("User:", username);      // From emit
    console.log("Extra:", extraInfo);    // From function call
})("Logged in from Chrome Browser");

An Immediately Invoked Function Expression (IIFE). It's defined and executed immediately with the extra parameter. It has access to both username (from outer scope) and extraInfo (its own parameter).

Emit the Event 
emitter.emit("login", "Nazeeha");

Triggers the "login" event and passes "Nazeeha" as the username parameter to all registered listeners.
